# CONTRIBUTING.md\n\n## Contributing to ParagraphLinks\n\nThank you for your interest in contributing to the ParagraphLinks MediaWiki extension! This document provides guidelines for contributing to the project.\n\n## Development Setup\n\n### Prerequisites\n\n- MediaWiki 1.35.0 or higher\n- PHP 7.4 or higher\n- Node.js and npm (for linting and development tools)\n- Git\n\n### Setting Up Development Environment\n\n1. **Clone the repository:**\n   ```bash\n   cd /path/to/mediawiki/extensions/\n   git clone https://github.com/yourusername/mediawiki-extension-paragraphlinks.git ParagraphLinks\n   cd ParagraphLinks\n   ```\n\n2. **Install development dependencies:**\n   ```bash\n   npm install\n   ```\n\n3. **Enable the extension in MediaWiki:**\n   Add to your `LocalSettings.php`:\n   ```php\n   wfLoadExtension( 'ParagraphLinks' );\n   \n   // Development settings\n   $wgShowExceptionDetails = true;\n   $wgDevelopmentWarnings = true;\n   $wgMainCacheType = CACHE_NONE;\n   $wgCacheDirectory = false;\n   ```\n\n4. **Run MediaWiki update:**\n   ```bash\n   php /path/to/mediawiki/maintenance/update.php\n   ```\n\n## Code Standards\n\n### PHP Code\n\n- Follow [MediaWiki PHP coding conventions]\n- Use tabs for indentation\n- Add PHPDoc comments for all public methods\n- Follow PSR-12 naming conventions\n- All classes should have proper namespace declarations\n\n### JavaScript Code\n\n- Follow [MediaWiki JavaScript coding conventions]\n- Use ESLint with Wikimedia configuration (included)\n- Use modern ES6+ features when appropriate\n- Add JSDoc comments for functions\n- Use meaningful variable and function names\n\n### CSS Code\n\n- Follow [MediaWiki CSS coding conventions]\n- Use meaningful class names with `paragraphlinks-` prefix\n- Support accessibility features (high contrast, reduced motion)\n- Test on multiple browsers and screen sizes\n\n## Testing\n\n### Running Tests\n\n```bash\n# Run PHP unit tests\nphp /path/to/mediawiki/tests/phpunit/phpunit.php extensions/ParagraphLinks/tests/phpunit/\n\n# Run JavaScript linting\nnpm run lint\n\n# Fix linting issues automatically\nnpm run lint:fix\n```\n\n### Writing Tests\n\n- Write unit tests for all new functionality\n- Test both success and failure scenarios\n- Include edge cases in your tests\n- Mock external dependencies properly\n- Follow MediaWiki testing conventions\n\n## Submitting Changes\n\n### Pull Request Process\n\n1. **Fork the repository** on GitHub\n\n2. **Create a feature branch:**\n   ```bash\n   git checkout -b feature/your-feature-name\n   ```\n\n3. **Make your changes:**\n   - Write clear, concise commit messages\n   - Keep commits focused and atomic\n   - Add tests for new functionality\n   - Update documentation as needed\n\n4. **Test your changes:**\n   ```bash\n   # Run all tests\n   npm run lint\n   php /path/to/mediawiki/tests/phpunit/phpunit.php extensions/ParagraphLinks/\n   ```\n\n5. **Submit a pull request:**\n   - Provide a clear description of the changes\n   - Reference any related issues\n   - Include screenshots for UI changes\n   - Ensure all checks pass\n\n### Commit Message Format\n\nUse clear, descriptive commit messages:\n\n```\nAdd hover link icons to paragraphs\n\n- Implement automatic anchor generation\n- Add clipboard integration with fallback\n- Include mobile touch support\n- Add accessibility features\n\nFixes #123\n```\n\n## Code Review Guidelines\n\n### For Contributors\n\n- Be responsive to feedback\n- Make requested changes promptly\n- Ask questions if feedback is unclear\n- Test thoroughly before requesting review\n\n### For Reviewers\n\n- Be constructive and helpful\n- Explain the reasoning behind suggestions\n- Focus on code quality, security, and maintainability\n- Test the changes locally when possible\n\n## Security Considerations\n\n- Never process untrusted user input server-side\n- Sanitize all generated HTML content\n- Use secure clipboard APIs when available\n- Follow MediaWiki security best practices\n- Report security issues privately\n\n## Accessibility Requirements\n\n- Support keyboard navigation\n- Provide proper ARIA labels\n- Ensure sufficient color contrast\n- Support screen readers\n- Test with accessibility tools\n- Respect user preferences (reduced motion, high contrast)\n\n## Browser Support\n\n- Modern browsers (Chrome, Firefox, Safari, Edge)\n- Graceful degradation for older browsers\n- Mobile browser compatibility\n- Test on different screen sizes\n\n## Documentation\n\n- Update README.md for user-facing changes\n- Add inline code comments for complex logic\n- Update configuration documentation\n- Include examples in documentation\n- Keep changelog updated\n\n## Issue Reporting\n\n### Bug Reports\n\nWhen reporting bugs, please include:\n\n- MediaWiki version\n- PHP version\n- Browser and version\n- Steps to reproduce\n- Expected vs actual behavior\n- Console errors (if any)\n- Screenshots (if applicable)\n\n### Feature Requests\n\nFor feature requests, please include:\n\n- Clear description of the feature\n- Use case and motivation\n- Proposed implementation (if any)\n- Potential impact on existing functionality\n\n## Release Process\n\n1. Update version numbers in `extension.json` and `package.json`\n2. Update `CHANGELOG.md` with release notes\n3. Create a GitHub release with tag\n4. Update MediaWiki extension registry (if applicable)\n\n## Getting Help\n\n- Create an issue on GitHub for questions\n- Check existing issues and documentation first\n- Provide context and examples when asking questions\n- Be patient and respectful\n\n## License\n\nBy contributing, you agree that your contributions will be licensed under the same GPL-2.0-or-later license as the project.\n\n[MediaWiki PHP coding conventions]: https://www.mediawiki.org/wiki/Manual:Coding_conventions/PHP\n[MediaWiki JavaScript coding conventions]: https://www.mediawiki.org/wiki/Manual:Coding_conventions/JavaScript\n[MediaWiki CSS coding conventions]: https://www.mediawiki.org/wiki/Manual:Coding_conventions/CSS
